ZaiSpace 在找一位 AI-native Design Engineer

我们是 ZaiSpace ，一个正在探索 AI 原生社交产品的早期团队。 我们相信，AI 不应该替代人与人之间的关系，而应该让真实连接变得更轻松、更自然、更有意思。现在我们正在用 self-evolving multi-agent 做个性化交互体验，尝试创造一个有生命力的社交空间。 公司已获得头部 VC 投资，并入选硅谷顶级孵化器 StartX 。团队成员来自字节、虾皮、P&G 、美元 VC ，毕业于 Stanford GSB 、UCSD HCI Lab 、复旦大学，分布在硅谷和上海。

目前我们还在产品探索期。也就是说：方向会变，原型会推倒重来，需求不会永远清晰。但这也意味着，你不会只是“接需求、写页面、下班”，而是会一起判断什么值得做、怎么做才真的好玩、好用、像一个有生命力的产品。

我们在找谁

我们想找一位 AI-native Design Engineer ，一起把 AI 社交产品从想法做成真实可用的体验。 这个岗位不太像传统前端，也不太像传统设计。我们更在意的是你有没有产品感、审美和判断力：你能不能看出一个功能有没有必要，一个页面有没有质感，一个交互是不是顺，一个产品是不是“像那么回事”。

前端能力是工具，不是门槛本身。我们不要求你是手写一切的资深前端，但希望你能熟练借助 AI coding 工具、设计工具和现成框架，把想法快速做出来、改到顺、推上线。 简单说，我们找的不是“会写页面的人”，而是一个能把产品体验做出来的人。 前期主要做 Web 。等产品方向更清楚后，你也可能参与客户端开发，比如移动端、跨端，或者其他更适合用户使用的产品形态。

你会做什么

  • 参与产品讨论，一起判断功能该不该做、怎么做更合理
  • 快速做出可用原型，把模糊想法推进到真实页面和真实交互
  • 借助 AI coding 工具完成 Web 端核心体验的设计实现、迭代和上线
  • 打磨 UI 和交互细节，包括布局、动效、加载态、空状态、异常状态、响应式适配
  • 把页面、接口、状态管理和业务逻辑串成完整体验
  • 在需要时参与工具和技术选型，选择适合当前阶段的方案
  • 产品方向明确后，可能参与移动端或跨端开发

我们希望你

  • 有很强的产品感，能判断什么值得做、怎么做更顺、哪里会让用户困惑
  • 有不错的审美和 UI 判断力，看到粗糙、不协调、不自然的地方会想改
  • 熟悉 AI 辅助开发，能用 Cursor / Claude Code / v0 / Lovable 等工具快速做原型和迭代
  • 不一定是传统资深前端，但需要能把 Web 页面、交互、接口和状态串成可用产品
  • 对细节敏感，在意空状态、加载态、动效、响应式、错误提示这些“产品质感”的部分
  • 执行力强，能从模糊需求里自己拆问题、找方案、往前推
  • 远程协作靠谱，能主动同步进度、问题和风险
  • 对 AI 产品、社交产品、新交互形态有兴趣

加分项

  • 做过从 0 到 1 的产品、Demo 或 side project
  • 会用 Figma ，或者能和设计工具顺畅协作
  • 熟悉 AI 辅助开发 / vibe coding
  • 有开源项目经验
  • 喜欢研究交互、动画、视觉细节
  • 看到一个丑页面会忍不住想动手救一下
  • 对“差不多能跑了”这句话有天然警惕
